# Cyber Incident Response & Security Best Practices

## Overview

This project was completed as part of a Cyber Security Internship task focused on understanding cyber incident response and organizational security practices. The objective was to learn how organizations identify, contain, eradicate, and recover from cyber security incidents while improving future security measures.

## Incident Response Lifecycle

### 1. Identification

The first step is detecting and confirming a cyber security incident. Security teams analyze alerts, logs, and unusual activities to determine whether an incident has occurred.

### 2. Containment

Once an incident is identified, immediate actions are taken to prevent it from spreading. Affected systems may be isolated from the network to minimize damage.

### 3. Eradication

The root cause of the incident is identified and removed. This may include deleting malware, closing security gaps, and removing unauthorized access.

### 4. Recovery

Systems and services are restored to normal operation. Organizations verify that affected systems are functioning properly before returning them to production.

### 5. Lessons Learned

After recovery, the incident is reviewed to identify weaknesses and improve future security controls and response procedures.

## Case Study: WannaCry Ransomware Attack

The WannaCry ransomware attack occurred in 2017 and affected organizations across the world. The malware exploited unpatched Windows systems and encrypted files, demanding ransom payments from victims.

### Impact

* Disruption of critical services
* Operational downtime
* Financial losses
* Reduced productivity

### Lessons Learned

* Keep systems updated
* Maintain regular backups
* Implement strong security controls
* Prepare incident response procedures

## Hypothetical Incident Response Plan

Scenario: An employee receives a phishing email and downloads a malicious attachment.

Response Plan:

1. Identify affected devices and users.
2. Disconnect infected systems from the network.
3. Remove malicious software.
4. Restore affected systems from backups.
5. Monitor for further suspicious activity.
6. Conduct employee awareness training.

## Security Best Practices Checklist

✓ Strong Password Policies

✓ Regular Software Updates

✓ Security Awareness Training

✓ Secure Data Backups

✓ Antivirus and Endpoint Protection

✓ Access Control Policies

✓ Log Monitoring and Auditing
